# Предупреждения и ограничения
- Обязательно использование видимого gui браузера, продублировано в [pytest.ini](test/pytest.ini) `addopts = --headed`
- Для каждой новой сессии необходимо проходить капчу после передачи запроса в поисковик. Капча проходится вручную пользователем!

# Шаг 0 - Установка пакетов
- Установить библиотеки из файла [requirements-pw.txt](requirements-pw.txt)
- Установить недостающий функционал playwright командой `playwright install`. [Инструкция (playwright.dev)](https://playwright.dev/python/docs/intro)
- Установить allure. [Инструкция (allurereport.org)](https://allurereport.org/docs/v2/install/) 

# Шаг 1 - Тесты
Тесты размещены в директории [test](test)  
Реализованы следующие тесты на основе функцоинала google.com: 
- Проверка текстового поиска [test_text_search.py](test/test_text_search.py)
- - Логика проверки: ищем факт совпадения выдачи поиска с ожидаемым результатом. Ожидаемый результат: результат пересечения существенен.
- Проверка поиска по изображению [test_image_search.py](test/test_image_search.py)
- - Логика проверки: передаём скриншот через всплывающее окно "Поиск по картинке". Ожидаемый результат: наличие Data Lens интерфейса в результатах поиска.
- Проверка позитивного сценария [test_positive_scenario.py](test/test_positive_scenario.py)
- - Логика проверки: пересечение трёх результатов: результат поиска с первой страницы с переданного запроса; результат поиска с первой страницы, после перехода на под-страницу (новости\картинки\др) и обратно; ожидаемым результатом. Ожидаемый результат: результат пересечения существенен.
- Проверка негативного сценария [test_negative_scenario.py](test/test_negative_scenario.py)
- - Логика проверки (поиск по изображению): Передача файлов некорректного расширения, передача некорректной ссылки на изображение. Ожидаемый результат: ссылка страницы остается неизменной.
- - Логика проверки (поиск по тексту): Передача запроса на 3000 символов. Ожидаемый результат: в поле запроса будет помещено только максимальное число символов (2048).

После запуска тестов генерируются вспомогательные директории `allure-results` и `artifacts`

# Шаг 2 - генерация и чтение репортов `allure`
1. Необходимо перейти в директорию, где размещена папка `allure-report`
2. Сгенерировать allure-отчет
- `allure generate allure-results -o allure-report --clean`
3. Открыть сгенерированный отчет
- `allure open allure-report`
4. В интерфейсе проанализировать исполнение тестов, вложенные скриншоты, скачать трейс-архив для playwright, если необходимо.
Документация: [How to view report (allurereport.org)](https://allurereport.org/docs/v2/view-report/)

# Шаг 3 - открытие trace.zip через `playwright`
1. Скачать trace-архив с allure-отчета или открыть директорию `artifacts`
2. Открыть trace-архив `playwright show-trace 'путь до trace-архива'`
Документация: [Trace viewer (playwright.dev)](https://playwright.dev/python/docs/trace-viewer)
